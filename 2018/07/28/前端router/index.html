Vue router

前端路由

前端路由的概念是什么？

不同的 url 地址，能让页面上有不同的显示内容（vue 组件）。并且还能处理浏览器前进后退功能。

SPA 的概念

SPA - single page application (单页面应用程序) 项目只有一个 html
MPA - multer page application (多页面应用程序) 项目有多个 html

多页面之间的切换是通过 a 标签，浏览器会刷新，
单页面之间的切换是通过 替换页面中某个位置的组件来实现的。浏览器不会刷新

vue-router 插件

使用 vue 路由的步骤

  1. 页面引入这个文件 script
  2. 根据需求与 PSD 分析得出页面中哪个位置是会发生变化的。在此位置放置路由的坑（路由的视图）router-view 组件
  3. 写好需要切换的页面组件
  4. 配置路由的规则
    http://localhost:8080/#/home -> Home 组件
    http://localhost:8080/#/list -> List 组件
    http://localhost:8080/#/about -> About 组件
  5. 如果需要路由的切换，可以使用 a 标签(需要注意写法)， 或者使用 router-link 组件

vue-router 中包含有两种路由模式

  • hash (默认)
  • history

new VueRouter({
mode: ‘history’,
routes: []
})

比较两种模式的区别

  1. url 地址表现不同，hash 模式在 url 地址有 # 号
  2. 实现原理不同
    1. hash onHashChange
    2. history HTML5 新增的 history 相关的 api
      history.pushState()
      history.replaceState()
      window.onpopstate
  3. history 在上线之后放置到静态 web 服务器中，刷新会出现 404 的错误，需要后台配合。

导航守卫

在路由进行跳转的时候，能监听到，并做一些事情。

三大块：

  • 全局守卫
    • 全局前置 beforeEach
    • 全局后置 afterEach
  • 路由独享 在路由规则中设置
    • beforeEnter
  • 组件中独享 在组件的选项中设置
    • beforeRouteEnter 进入的时候
    • beforeRouteUpdate 更新的时候
    • beforeRouteLeave 离开的时候

守卫中的参数

  • to - 将要去的路由
  • from - 从哪里去的路由
  • next - 控制是否需要让他去
    1. 不调用 next(),或者 next(false) 路由将不能跳转
    2. 直接调用 next() ， 那么路由想去哪，就去哪
    3. next(‘/detail/apple’); 被打掉哪个路由里面去。

路由守卫的一些使用场景

  1. 如果是从 A->B 那么不然它去，其他页要去 B ok .

  2. 全局前置，判断 from 是不是 A,并且判断 to 是不是 B X

  3. beforeRouteLeave 写在 A 的组件上，判断 to 是不是 B

  4. A B C D E F G, A B 必须要登录才能进去。

  5. 全局判断 to 是否等于 A || B ,然后再判断有么有登录状态。 ✅

  6. 在 A B 的 beforeEnter 中，判断登录状态。
