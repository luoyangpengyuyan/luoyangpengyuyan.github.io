vue组件

组件：

  • 就是一个具有预定义选项的 vue 实例，一个组件也可以看成是一个 vue 的实例对象，new Vue 的时候有哪些选项可以配置，那么组件也有这些选项可以配置，（但是 el 这个选项不需要）

一、组件的创建与使用：

组件的创建

  1. 全局组件的创建
  • Vue.component(‘组件的名字’, 组件的选项对象)
  1. 局部组件的创建
  • 组件 或者 根组件(new Vue), 配置 components 选项

组件的使用

  • 将组件的名字当成自定义标签来使用即可。

组件的一些注意事项：

  1. 全局一定要在 new Vue 之前创建
  2. 组件的名字，不能是现有的 html 元素。
  3. 组件不需要 el 选项
  4. 组件必须要有 template 或者 render 选项。
  5. 组件的 data 选项必须是一个函数，返回对象的形式。
  6. 组件只能有一个根元素
  7. 组件的开始标签与闭合标签的内容默认是不会渲染出来的。
  8. 组件接收到的 props 不允许修改，会导致数据难以理解。（单向数据流）

非 props 特性： 在调用组件的时候可以写很多的属性，如果某个属性没有在这个组件中的 props 选项中定义，那么这个属性就叫做 非 props 特性

  1. 非 props 特性都有继承的特性。
  2. 继承的情况是替换，而 class 和 style 他们是 合并
  3. 可以通过一个选项来禁止非 props 特性的继承, class 与 style 除外 inheritAttrs
  4. 有时我们不需要非 prop 特性继承到组件根元素，但是又需要他继承到组件的某个元素上，这时 可以结合 $attrs 与 inheritAttrs 来使用

$attrs 他上面有得就是在调用组件的时候标签上写的那些特性

一些特殊的操作

1. $root      - 得到根组件的实例对象
2. $parent    - 得到父组件的实例对象
3. $children  - 得到子组件的实例对象
4. $refs      - 得到子组件的实例对象或者是当前组件内的某个dom元素
                前提是，你需要在要操作组件或元素身上设置 ref 这个属性

上面4中都不推荐去操作组件的实例对象, 推荐使用 $refs 去操作dom对象

1. 不要使用 id 或者 class 或者 标签名字的方式去修改组件的dom元素，会造成困扰

插槽 slot 坑

  • 它的作用是，能够将在组件的开始标签与结束标签中写的内容渲染到页面上。
    实现步骤：
  1. 组件是否需要插槽
  2. 如果需要，我这个插槽的内容需要渲染在组件的那个位置
  3. 要渲染在哪里，就在哪里写上个
  • 概念

    插槽的模板内容： 开始标签与结束标签中的内容

一、多个插槽，给坑取名字

  1. slot name 属性
  2. 给 插槽模板内容 设置 slot 属性，属性的值为 坑的名字

二、插槽的后备内容（默认内容）

一、生命周期：
vue 实例从创建到销毁，会有各种阶段，整个流程就是生命周期

二、生命周期函数 | 生命周期钩子函数
在生命周期的某个阶段会自动触发的函数

三、三大阶段

1. 挂载阶段
  beforeCreate
  created
  beforeMount
  mounted

2. 更新阶段
  beforeUpdate
  updated

3. 销毁阶段
  beforeDestory
  destoryed

1. beforeCreate       创建之前    数据的初始化工具
  不能拿到实例数据
2. created            创建完成
  异步请求
3. beforeMount        挂载之前
  不能拿到真实的dom树
4. mounted            挂载完成
  能拿到真实的dom树
5. beforeUpdate       更新之前
  保存之前的dom树的信息
6. updated            更新完成
  数据更新完成，并且真实的dom树也渲染完成
7. beforeDestroy      销毁之前
  能使用 this
  1. 将一些全局的事件监听的取消
  2. 计时器
8. destroyed          销毁完成
9. actived            激活
10. deactived         失活
